## Стратегия чункования

Выбрана стратегия **«один chunk = одна функция, метод или класс»**.

**Почему это оптимально?**

1. **Семантическая целостность** — функция является минимальной самодостаточной единицей логики в Python. В ней есть входные данные, действие и результат.

2. **Чёткие границы** — библиотека AST (Abstract Syntax Tree) предоставляет точные номера строк начала и конца каждой функции или класса. Можно легко вырезать нужный фрагмент.

3. **Оптимальный размер** — целый файл (500+ строк) даёт слишком много мусора. Одна строка кода не даёт контекста. А функция или класс — ровно столько, сколько нужно, чтобы понять смысл.

4. **Релевантность поиска** — пользовательские вопросы обычно звучат как «как работает функция login?» или «что делает метод get_user?». То есть вопросы уже нацелены на конкретные функции и методы.

## Формат идентификатора чанка (chunk_id)
{путь_к_файлу}:{имя}:{номер_строки_начала}
